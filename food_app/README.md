1. **简化的数据库设计**：使用SQLite数据库替代MySQL，避免了数据库配置的复杂性，同时保持了数据存储的功能。

2. **核心功能模块**：
   - 用户投稿：支持图文投稿，数据存入SQLite数据库

   - 自动发布：审核通过后自动生成Markdown文件并触发MkDocs重建

3. **文件结构**：
   - `app.py`：Flask应用的主文件，包含所有路由和核心功能
   - `templates/`：HTML模板文件，包含首页、投稿页面、登录页面和审核页面
   - `uploads/`：上传图片的存储目录
   - `food_review.db`：SQLite数据库文件

4. **核心算法**：
   - 审核状态流转：通过状态字段（pending/approved/rejected）实现状态管理
   - Markdown生成：根据投稿内容生成符合现有网站格式的Markdown文件
